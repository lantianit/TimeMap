# PRD-09：足迹地图 — 用户全域足迹可视化

## 1. 背景与目标

### 1.1 背景

当前用户主页（profile / user 页面）的"足迹亮点"卡片仅以文字列表形式展示用户去过的区域和照片数量。用户无法直观感受自己（或他人）的足迹分布全貌，缺少一个"一眼看到所有去过的地方"的沉浸式体验。

作为一款以"用照片标记时间，用地图丈量记忆"为核心理念的产品，足迹地图是用户感知产品价值的关键功能，直接影响用户留存和分享意愿。

### 1.2 目标

- 提供全域足迹地图视图，在地图上完整展示用户所有照片的地理分布
- 支持查看自己和他人的足迹（受可见性权限控制）
- 多级缩放体验：全国概览 → 城市聚合 → 街道级单张照片
- 提供足迹统计摘要，增强成就感
- 为后续"足迹分享海报"、"年度足迹报告"等功能奠定基础

### 1.3 入口

| 入口位置 | 触发方式 | 说明 |
|---------|---------|------|
| 他人主页 - 足迹亮点卡片 | 点击"查看全部 ▸" | 查看该用户的公开足迹 |
| 我的主页 - 足迹亮点卡片 | 点击"查看全部 ▸" | 查看自己的全部足迹 |
| 我的主页 - 统计栏"区域"数字 | 点击区域数字 | 快捷入口，同上 |

## 2. 核心功能

### 2.1 足迹地图视图

#### 2.1.1 初始视角

- 进入页面后，地图自动调整缩放级别和中心点，使所有足迹标记都在可视范围内
- 使用 `mapCtx.includePoints()` 实现自适应视角
- 若用户只有一个区域的照片，默认缩放到 scale=12（城市级别）
- 若用户没有任何照片，显示空状态引导

#### 2.1.2 多级标记策略

根据地图缩放级别（scale）采用不同的标记展示策略：

| 缩放级别 | 展示方式 | 说明 |
|---------|---------|------|
| scale ≤ 8（全国/省级） | 城市气泡标记 | 显示城市名 + 照片总数，如"杭州 · 14张" |
| 8 < scale ≤ 12（城市级） | 区县气泡标记 | 显示区县名 + 照片数，如"西湖区 · 8张" |
| scale > 12（街道级） | 照片缩略图标记 | 复用现有地图页的缩略图卡片标记样式 |

气泡标记设计：
- 圆形气泡，白色背景，绿色边框
- 内部显示照片数量（大字）+ 地名（小字）
- 气泡大小随照片数量适度缩放（最小 80rpx，最大 140rpx）
- 点击气泡：地图平滑放大到该区域中心，切换到下一级展示

#### 2.1.3 照片缩略图标记（街道级）

- 复用现有地图页的 `photo-callout` 样式
- 同一位置多张照片聚合，显示数量角标
- 点击标记跳转到照片详情页

### 2.2 顶部信息栏

固定在页面顶部，毛玻璃背景，显示：

```
┌─────────────────────────────────────┐
│  [头像] xxx 的足迹                    │
│  共 24 张照片 · 覆盖 7 个区域 · 3 个城市  │
└─────────────────────────────────────┘
```

- 查看自己时显示"我的足迹"
- 查看他人时显示"xxx 的足迹"
- 统计数据：照片总数、覆盖区域数、覆盖城市数

### 2.3 底部足迹摘要面板

可上滑展开的底部面板（类似地图 App 的 POI 详情面板），默认收起只露出摘要行：

#### 收起状态（默认）
```
┌─────────────────────────────────────┐
│  ─── （拖拽指示条）                    │
│  📍 足迹覆盖 3 个城市 7 个区域          │
└─────────────────────────────────────┘
```

#### 展开状态（上滑）
```
┌─────────────────────────────────────┐
│  ─── （拖拽指示条）                    │
│  📍 足迹覆盖 3 个城市 7 个区域          │
│                                     │
│  杭州市                    14张  ▸    │
│    西湖区          8张               │
│    上城区          6张               │
│  湖州市                     6张  ▸   │
│    吴兴区          6张               │
│  承德市                     4张  ▸   │
│    承德县          4张               │
└─────────────────────────────────────┘
```

- 按城市分组，城市内按区县列出
- 每行显示区域名 + 照片数量
- 点击城市行：地图飞到该城市中心并放大
- 点击区县行：地图飞到该区县中心并放大

### 2.4 空状态

- 自己无照片："你还没有足迹记录，去地图上传第一张照片吧" + 跳转按钮
- 他人无照片："TA 还没有公开的足迹记录"

### 2.5 可见性权限

| 查看者身份 | 可见的照片范围 |
|-----------|-------------|
| 本人 | 全部照片（含仅自己可见） |
| 互关好友 | visibility=2（所有人）+ visibility=1（互关可见） |
| 陌生人 | 仅 visibility=2（所有人可见） |
| 未登录 | 仅 visibility=2（所有人可见） |

## 4. 技术方案

### 4.1 新增后端接口

#### `GET /api/photo/footprint`

返回指定用户的所有照片坐标及区域聚合数据。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| targetUserId | Long | 否 | 目标用户ID，不传则查自己 |

**响应数据：**

**性能约束：**
- 照片列表最多返回 500 条（按 photo_date DESC 取最新的）
- 聚合数据在 SQL 层完成，不在应用层遍历
- 城市/区县的中心坐标取该区域内照片的平均经纬度

**可见性过滤：**
- 查看自己：返回全部照片（含 visibility=0）
- 查看他人：根据登录状态和互关关系过滤（复用现有 visibility 过滤逻辑）

### 4.2 前端新增页面

新增页面：`pages/footprint/footprint`

#### 页面参数

| 参数 | 说明 |
|------|------|
| userId | 目标用户ID（不传则查自己） |
| nickname | 用户昵称（用于顶部显示） |
| avatarUrl | 用户头像URL |

#### 核心逻辑

```
onLoad:
  1. 请求 /api/photo/footprint?targetUserId=xxx
  2. 用 summary.cityGroups 构建城市级气泡标记
  3. 用 mapCtx.includePoints() 自适应视角
  4. 监听 onRegionChange，根据 scale 切换标记级别

onRegionChange:
  scale ≤ 8  → 渲染城市气泡（cityGroups 数据）
  8 < scale ≤ 12 → 渲染区县气泡（当前可视范围内的 districts）
  scale > 12 → 渲染照片缩略图（当前可视范围内的 photos）

气泡点击:
  → mapCtx.moveToLocation + 放大 scale
  → 触发 onRegionChange → 自动切换到下一级标记
```

#### 标记渲染策略

由于微信小程序 map 组件的 `customCallout` 不支持动态切换，采用以下方案：

- 维护三套标记数据：`cityMarkers`、`districtMarkers`、`photoMarkers`
- `onRegionChange` 时根据 scale 切换 `this.setData({ markers: xxxMarkers })`
- 照片缩略图标记复用现有 `buildThumbUrl()` 函数生成缩略图 URL
- 照片标记按经纬度聚合（复用现有 `loadNearbyPhotos` 的聚合逻辑）

### 4.4 性能优化

| 优化点 | 方案 |
|--------|------|
| 标记数量控制 | 照片最多 500 张；城市/区县气泡通常 < 50 个 |
| 缩略图加载 | 使用 `buildThumbUrl()` 追加 `imageMogr2/thumbnail/216x200` |
| 标记切换 | 仅在 scale 跨越阈值时切换，避免频繁 setData |
| 可视范围过滤 | 街道级标记只渲染当前可视范围内的照片（前端过滤） |
| 数据缓存 | footprint 数据在页面生命周期内缓存，缩放切换不重复请求 |
