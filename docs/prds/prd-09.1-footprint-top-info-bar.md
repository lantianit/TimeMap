# PRD-09.1：足迹地图 — 顶部信息栏改造（替代底部面板）

## 1. 背景

当前足迹地图页底部有一个可上滑展开的摘要面板，显示"📍 5张照片 · 2个城市 · 2个区域"。存在以下问题：

- 底部面板遮挡地图下方区域，影响南方城市的气泡点击
- 面板展开/收起的交互较重，用户使用率低
- 信息是静态的，不会随用户在地图上的操作而变化，缺乏联动感

## 2. 目标

将底部面板移到顶部，改为一个轻量的上下文信息栏，核心特性：

- 随用户操作动态变化内容，形成"地图操作 ↔ 信息栏联动"的体验
- 不遮挡地图主体区域
- 去掉展开/收起的重交互，信息一目了然

## 3. 交互设计

### 3.1 信息栏位置与样式

固定在导航栏下方，毛玻璃胶囊样式，水平居中：

```
┌──────────────────────────────────────┐
│  <  我的足迹              ·· ⊙       │  ← 系统导航栏
├──────────────────────────────────────┤
│     ┌──────────────────────┐         │
│     │  📍 5张 · 2城市 · 2区域 │         │  ← 信息胶囊（顶部居中）
│     └──────────────────────┘         │
│                                      │
│            全屏地图                    │
│                                      │
│                              [↻]     │  ← 重置按钮（点击城市后出现）
│                                      │
└──────────────────────────────────────┘
```

样式规范：
- 圆角胶囊：`border-radius: 32rpx`
- 背景：`rgba(255,255,255,0.92)` + `backdrop-filter: blur(16px)`
- 阴影：`0 2rpx 12rpx rgba(0,0,0,0.08)`
- 内边距：`12rpx 28rpx`
- 字号：`24rpx`，颜色 `#333`
- z-index: 200，不随地图滚动

### 3.2 三种状态的内容变化

信息栏内容随用户操作在三种状态间切换：

#### 状态一：全国概览（默认）

用户刚进入页面，或点击重置按钮后：

```
📍 5张照片 · 2个城市 · 2个区域
```

显示用户的全局足迹统计。

#### 状态二：城市级

用户点击某个城市气泡后，信息栏切换为该城市的统计：

```
📍 武汉市 · 4张照片 · 1个区域
```

显示当前聚焦城市的名称、照片数、区域数。

#### 状态三：区县级

用户从底部面板（已移除）或从城市级继续点击某个区县后：

```
📍 武昌区 · 4张照片
```

显示当前聚焦区县的名称和照片数。

### 3.3 状态切换规则

| 用户操作 | 信息栏变化 |
|---------|-----------|
| 进入页面 | 状态一（全国概览） |
| 点击城市气泡 | 切换到状态二（该城市信息） |
| 点击区县气泡（如果有） | 切换到状态三（该区县信息） |
| 点击右上角重置按钮 ↻ | 回到状态一（全国概览） |
| 手动缩小地图到 scale ≤ 9 | 回到状态一（全国概览） |

### 3.4 切换动效

信息栏内容切换时，使用淡入过渡：
- 文字内容变化时，整个胶囊做一次轻微的 `opacity 0→1` 过渡（200ms）
- 胶囊宽度随内容自适应，无需固定宽度

## 4. 移除的内容

- 移除底部摘要面板（`.panel` 及相关 DOM、样式、JS 逻辑）
- 移除面板的展开/收起交互（`panelExpanded`、`onPanelToggle`、`onPanelTouchStart/Move/End`）
- 移除面板中的城市/区县列表点击跳转（`onPanelCityTap`、`onPanelDistrictTap`）

> 城市/区县的跳转功能由地图上的气泡点击承担，不再需要列表入口。

## 5. 数据结构

复用现有 `summary` 数据，无需新增接口。信息栏所需数据：

```javascript
// 全国概览
{ level: 'overview', text: '📍 5张照片 · 2个城市 · 2个区域' }

// 城市级 — 从 summary.cityGroups[i] 取
{ level: 'city', text: '📍 武汉市 · 4张照片 · 1个区域' }

// 区县级 — 从 summary.cityGroups[i].districts[j] 取
{ level: 'district', text: '📍 武昌区 · 4张照片' }
```

新增 data 字段：
- `infoBarText`: 信息栏显示文本
- 移除：`panelExpanded`、`cityBubbles`（面板相关）

## 6. 边界情况

| 场景 | 处理 |
|------|------|
| 只有1个城市1个区域 | 全国概览显示"📍 X张照片 · 1个城市 · 1个区域" |
| 照片无 district 信息 | 城市级显示时区域数不含未标注的 |
| 文字过长 | 胶囊最大宽度 `calc(100vw - 80rpx)`，超出省略 |

## 7. 实现范围

| 改动 | 说明 |
|------|------|
| `footprint.wxml` | 移除底部面板 DOM，新增顶部信息胶囊 |
| `footprint.wxss` | 移除面板样式，新增信息胶囊样式 |
| `footprint.js` | 移除面板逻辑，新增 `infoBarText` 状态管理，在城市/区县点击和重置时更新 |
